## About Us
We are a passionate data team focused on educational analytics. We work closely with student data to develop innovative solutions that support learning and teaching at Vrije Universiteit Amsterdam. Our projects include:

- Data Pipelines: Efficiently processing and managing large datasets.
  - [canvas-analytics](https://github.com/vusaverse/canvas-analytics); pipeline of Canvas LMS
  - [evalytics](https://github.com/vusaverse/evalytics); pipeline for evluation data
- R Packages: Creating reusable code for data analysis and visualization.
  - [vvauditor](https://github.com/vusaverse/vvauditor); data audits (CRAN)
  - [vvcanvas](https://github.com/vusaverse/vvcanvas); Canvas LMS api (CRAN)
  - [vvconverter](https://github.com/vusaverse/vvconverter); data convertions (CRAN)
  - [vvfiller](https://github.com/vusaverse/vvfiller); fill data (CRAN)
  - [vvmover](https://github.com/vusaverse/vvmover); read write data (CRAN)
  - [vvprojector](https://github.com/vusaverse/vvprojector); setup projects
  - [vvsculptor](https://github.com/vusaverse/vvsculptor); models and joins (CRAN)
  - [vvshiny](https://github.com/vusaverse/vvshiny); build shiny apps with templates (CRAN)
  - [vvtableau](https://github.com/vusaverse/vvtableau); Tableau api (CRAN)
  - [vvtermtime](https://github.com/vusaverse/vvtermtime); Semestry TermTime api (CRAN)
- R Shiny Apps: Developing interactive web applications for data exploration.
  - [vvdoctor](https://github.com/vusaverse/vvdoctor); Statistics shiny app (CRAN)
- Machine Learning Models: Implementing predictive analytics to drive insights from educational data.
  - [conversion_model](https://github.com/vusaverse/converion_model); conversion of students
- Other repositories
  - [docker-rocker](https://github.com/vusaverse/docker-rocker); Dockerized RStudio instance
  - [data_asset_catalog](https://github.com/vusaverse/data_asset_catalog); data assets
  - [snippets](https://github.com/vusaverse/snippets); rstudio snippets
  - [vusaverse.github.io](https://github.com/vusaverse/vusaverse.github.io); website built in Quarto

## Contribution Guidelines
We welcome contributions from the community! If you're interested in collaborating with us, please consider the following:

- Fork the Repository: Start by forking the repository you want to contribute to.
- Create a Feature Branch: Use a descriptive name for your branch.
- Make Your Changes: Implement your feature or fix.
- Submit a Pull Request: Once you're ready, submit a pull request detailing your changes.
